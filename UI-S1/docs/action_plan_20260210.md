## 核心洞察：我们的真正问题是什么？

### 关键发现重新审视

```
┌─────────────────────────────────────────────────────────────────┐
│                        ROOT CAUSE 分析                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  问题1: 奖励函数崩塌 (⭐⭐⭐⭐⭐ 最关键)                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ critic/score/mean: 0.10 (常数)                          │   │
│  │ critic/advantages/mean: 0.00                            │   │
│  │ → 当 advantages = 0 时，policy gradient = 0             │   │
│  │ → 模型无法学习！这不是"学习缓慢"，而是"无法学习"         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  问题2: 格式退化 (⭐⭐⭐⭐ 重要)                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 训练数据: 0 格式错误                                     │   │
│  │ 训练后: 10.6% 格式错误 (+360%)                           │   │
│  │ → 模型学到了"错误的东西"                                 │   │
│  │ → 说明 KL 惩罚失效，策略偏离了预训练模型的正确约束        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  问题3: 数据不平衡 (⭐⭐⭐)                                     │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ click: 51.7% → 84.1% (+32.4%)                           │   │
│  │ terminate: 15.3% → 0.3% (-15.0%)                        │   │
│  │ → 这是症状，不是根本原因                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 关键洞察

**洞察 1**: 当 `advantages = 0` 时，即使调整其他所有参数，模型都无法学习。这是**第一优先级**。

**洞察 2**: 格式退化说明模型正在"遗忘"预训练知识。需要立即加强 KL 惩罚。

**洞察 3**: Action 分布偏移是奖励函数崩塌的结果，而不是原因。如果奖励正确，分布会自动纠正。

### 阶段 1: 快速修复 (2-3天)

**目标**: 用最简单、最安全的方式，验证"修复奖励 + 加强 KL"是否有效

#### 实验 1.1: 奖励函数修复 ⭐⭐⭐⭐⭐

**假设**: 当前奖励函数导致所有样本获得相似分数

**修改**:
```yaml
# 在配置文件中调整
reward:
  # 基础奖励
  type_match_weight: 0.3
  detail_match_weight: 0.7

  # 步骤权重（早期步骤更重要）
  use_step_weight: true
  step_weight_decay: 0.1

  # Action 类型权重
  action_importance:
    open: 2.0
    type: 1.5
    click: 1.0
    swipe: 1.0
    terminate: 1.2
```

**验证指标**:
- `critic/score/std` 应该 > 0.05
- `critic/advantages/std` 应该 > 0.1
- 训练曲线应该有真实的学习信号

---

#### 实验 1.2: KL 惩罚加强 ⭐⭐⭐⭐

**假设**: KL = 0.001 太弱，无法防止策略偏离

**修改**:
```yaml
# LoRA 配置
actor_rollout_ref.actor.optim.kl_loss_coef: 0.1  # 增加 100x

# 或者更保守
actor_rollout_ref.actor.optim.kl_loss_coef: 0.05
```

**验证指标**:
- `actor/ppo_kl` 应该保持在 < 0.05
- 格式错误率应该下降
- Action 分布应该更接近训练数据
